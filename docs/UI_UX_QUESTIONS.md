# UI_UX_QUESTIONS.md — Deebi UI/UX 질의응답

이 문서는 UI/UX를 개발자와 함께 확정하기 위한 질문 목록이다. 답변이 확정되면 `docs/DECISIONS.md`에 한 줄로 기록하고 `docs/UI_UX.md`를 업데이트한다.

## 0. 현재까지 확정된 답변

- 서비스명은 `Deebi`다.
- Deebi는 픽셀 캐릭터 기반의 가벼운 방치형 작업/놀이 공간이다.
- 기본 공간은 **내 방**, 친구와 함께할 때는 **공동 룸**으로 들어간다.
- 내 방은 꾸미기 중심 개인 방으로 설계한다.
- 내 방에서 공동 룸으로 이동하는 대표 오브젝트는 **문**이다.
- 공동 룸은 큰 작업 테이블 중심으로 설계한다.
- 공동 룸 멤버 수 상한은 **8명**이다.
- 공동 룸 멤버 배치는 인원 수에 따라 자동 변경한다.
- 공동 룸 멤버 상태는 작은 라벨과 테이블 오브젝트/이펙트 조합으로 표현한다.
- 대시보드는 `내 방` / `함께하는 방` 상단 탭 구조다.
- 내 방 데스크톱 레이아웃은 큰 방 스테이지 + 오른쪽 관리 패널 + 하단 인벤토리 구조다.
- MVP에는 간단한 방치 모드를 포함한다.
- 방치 모드는 내 방 + 친구 상태 미니 신호 형태다.
- 방치 모드의 친구 상태 미니 신호는 문/창문/전광판 같은 방 안 오브젝트로 표현한다.
- 캐릭터 기본 해상도는 **32x32**다.
- 캐릭터 최종 디자인과 상태별 애니메이션 디자인은 개발자가 별도로 제공한다.
- 캐릭터 애니메이션은 상태별 idle 중심이며, 강도는 **살짝 살아 있는 정도**다.
- 권장 에셋 방식은 상태별 스프라이트시트 PNG + manifest JSON이다.
- Deebi의 말투는 **살짝 친근하게** 간다.
- 디비의 말풍선/시스템 대사는 **내 방에서만** 사용한다.
- 공동 룸에서는 디비가 끼어들지 않고, 친구들끼리 대화하는 공간으로 둔다.
- 공동 룸 텍스트 채팅은 MVP에 포함한다.
- 공동 룸 채팅은 룸 멤버 전용 짧은 텍스트 채팅이며, DM/전역 채팅/신고/차단/파일 첨부/읽음 표시는 MVP 제외다.
- 공동 룸 채팅 배치는 데스크톱 오른쪽 패널, 모바일 하단 시트로 확정한다.
- 기본 색상 분위기는 밝은 픽셀 방이다. warm ivory/cream 기반에 green/blue 포인트로 시작한다.
- 전체 UI/UX는 픽셀 그래픽 감성으로 구현한다.
- 앱 기본 폰트는 PF스타더스트로 확정한다.
- 랜딩 핵심 문장은 `각자 코딩해도, 같은 방에 있는 것처럼.`이다.
- 랜딩은 짧은 소개형 + 공동 룸 미리보기 조합으로 설계한다.
- 광고는 보류한다. MVP에서는 실제 SDK를 넣지 않고 구조적으로만 예약한다.
- MVP GitHub OAuth scope는 `read:user`만 사용한다.
- MVP 커밋 집계는 공개 PushEvent 기준으로 시작하고, 비공개 커밋 집계는 후속 확장으로 둔다.

## 6. 권한/신뢰

15. 비공개 커밋 집계를 위해 `repo` 권한을 요청해도 되는가?
    - **확정:** MVP에서는 요청하지 않는다. `read:user`만 사용하고 공개 활동으로 시작한다.
    - 후속 확장 후보: 사용자가 선택하게 하거나 GitHub App 전환을 검토한다.
