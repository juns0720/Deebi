# P09-T06 — 룸 권한, 제한, QA

## 목표

룸 생성/참여/나가기/채팅 흐름의 권한과 제한을 점검하고 Phase 09를 마무리한다.

## 변경 범위

- **해도 됨:** room 보안 수정, edge case 테스트, QA notes
- **하면 안 됨:** 친구 시스템, DM, 신고/차단, 룸 삭제 추가

## 작업

- 8명 상한, 중복 참여, 비멤버 접근을 확인한다.
- 메시지 길이와 권한을 확인한다.
- 룸 나가기 후 권한이 제거되는지 확인한다.
- MVP 제외 기능이 들어가지 않았는지 확인한다.
- Phase 09 결과를 기록한다.

## 완료 조건

- [ ] 8명 초과 참여가 거부된다.
- [ ] 중복 참여 row가 생기지 않는다.
- [ ] 비멤버는 룸 상세/메시지 접근이 거부된다.
- [ ] 룸 나가기 후 메시지 조회/작성이 거부된다.
- [ ] `npm run lint`, `npm run typecheck`, `npm run build`가 통과한다.

## 개발자 테스트

1. 8명 초과 참여를 시도한다.
2. 비멤버로 room API와 message API를 호출한다.
3. 룸 나가기 후 다시 접근해 거부되는지 확인한다.
