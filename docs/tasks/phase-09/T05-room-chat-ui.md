# P09-T05 — 공동 룸 채팅 UI와 갱신

## 목표

공동 룸에서 방과 캐릭터를 가리지 않는 채팅 UI를 구현한다.

## 변경 범위

- **해도 됨:** room chat panel, mobile bottom sheet, polling/manual refresh, input validation UI
- **하면 안 됨:** DM, 멘션, 파일 첨부, 신고/차단, 읽음 표시

## 작업

- 데스크톱 오른쪽 320~360px 채팅 패널을 만든다.
- 모바일 하단 시트 채팅을 만든다.
- 최근 메시지, 빈 상태, 오류 상태를 표시한다.
- 1~300자 입력 검증을 UI에 표시한다.
- MVP 수준의 짧은 폴링 또는 수동 갱신을 연결한다.
- 디비 말풍선이 공동 룸 채팅에 나오지 않게 한다.

## 완료 조건

- [ ] 데스크톱에서 채팅이 오른쪽 패널에 있다.
- [ ] 모바일에서 채팅이 하단 시트로 열린다.
- [ ] 긴 메시지와 긴 login이 레이아웃을 깨지 않는다.
- [ ] 입력 오류 상태가 보인다.
- [ ] 디비 말풍선은 공동 룸에 없다.

## 개발자 테스트

1. 데스크톱에서 룸 채팅을 확인한다.
2. 모바일 폭에서 하단 시트 열림/닫힘을 확인한다.
3. 빈 메시지와 300자 초과 메시지를 입력해 본다.
