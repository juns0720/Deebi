# P09-T01 — 룸 생성, 참여, 목록 API

## 목표

사용자가 룸을 만들고 코드로 참여하며, 자신이 참여한 룸 목록을 볼 수 있게 하는 API 기반을 만든다.

## 변경 범위

- **해도 됨:** `GET /api/rooms`, `POST /api/rooms`, `POST /api/rooms/join`, 내 룸 목록 조회, room code 생성
- **하면 안 됨:** 공동 룸 화면, 채팅 API, 친구 요청/수락

## 작업

- room code 생성 함수를 만든다.
- 내 룸 목록 조회 API `GET /api/rooms`를 구현한다.
- 룸 생성 API를 구현한다.
- 생성자를 첫 멤버로 추가한다.
- 코드 참여 API를 구현한다.
- 내가 참여한 룸 목록 조회 구조를 만든다.
- 8명 상한 검증을 둔다.

## 완료 조건

- [ ] 로그인 사용자가 룸을 만들 수 있다.
- [ ] 로그인 사용자가 참여 중인 룸 목록을 조회할 수 있다.
- [ ] 생성자가 자동으로 첫 멤버가 된다.
- [ ] code로 룸에 참여할 수 있다.
- [ ] 이미 참여한 룸에 중복 row가 생기지 않는다.
- [ ] 8명 초과 참여 시 `ROOM_FULL` 오류를 반환한다.

## 개발자 테스트

1. 로그인 사용자로 룸을 만든다.
2. 생성된 room code를 확인한다.
3. 같은 사용자로 다시 join해 중복이 없는지 확인한다.
4. 8명 초과 mock 또는 DB 상태에서 join이 거부되는지 확인한다.
