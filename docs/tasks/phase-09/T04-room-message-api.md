# P09-T04 — 룸 메시지 API

## 목표

같은 룸 멤버끼리만 최근 메시지를 조회하고 1~300자 텍스트 메시지를 보낼 수 있게 한다.

## 변경 범위

- **해도 됨:** `GET /api/rooms/[code]/messages`, `POST /api/rooms/[code]/messages`, validation
- **하면 안 됨:** DM, 전역 채팅, 메시지 수정/삭제, 파일 첨부, 읽음 표시

## 작업

- 최근 메시지 50개 조회 API를 만든다.
- 메시지 작성 API를 만든다.
- 룸 멤버십을 검증한다.
- trim 후 1~300자 제약을 적용한다.
- 오류 코드를 API 계약과 맞춘다.

## 완료 조건

- [ ] 룸 멤버는 최근 메시지를 볼 수 있다.
- [ ] 룸 멤버는 메시지를 작성할 수 있다.
- [ ] 룸 멤버가 아닌 사용자는 조회/작성할 수 없다.
- [ ] 빈 메시지와 300자 초과 메시지는 저장되지 않는다.
- [ ] 응답에 token, SQL, stack trace가 없다.

## 개발자 테스트

1. 룸 멤버로 메시지를 작성한다.
2. 최근 메시지 목록에 나타나는지 확인한다.
3. 룸 멤버가 아닌 사용자로 API 접근이 거부되는지 확인한다.
