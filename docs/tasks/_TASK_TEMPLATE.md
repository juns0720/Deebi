# PNN-TNN — Task 이름

## 목표

이 task가 끝났을 때 사용자와 시스템 관점에서 무엇이 가능해지는지 한두 문장으로 쓴다.

## 변경 범위

- **해도 됨:** 이 task에서 수정 가능한 파일, 모듈, 문서 범위를 적는다.
- **하면 안 됨:** 같은 phase의 다른 task 소관이거나 MVP 범위 밖인 작업을 적는다.

## 작업

- 구현해야 할 일을 기능 단위로 적는다.
- 완료 후 눈으로 확인 가능한 결과를 남긴다.

## 완료 조건

- [ ] 실행 또는 눈으로 검증 가능한 조건을 적는다.
- [ ] 자동 검증 명령이 있다면 포함한다.
- [ ] 문서 갱신 조건이 있다면 포함한다.

## 개발자 테스트

1. 개발자가 직접 따라 할 절차를 순서대로 적는다.
2. 기대 결과를 함께 적는다.
