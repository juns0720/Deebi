# P10-T02 — 랜딩과 dashboard polish

## 목표

랜딩과 dashboard의 정보 위계, 카피, 상태 화면을 실제 MVP 톤에 맞게 다듬는다.

## 변경 범위

- **해도 됨:** landing/dashboard UI polish, copy, empty/loading/error state
- **하면 안 됨:** 새 제품 기능, 결제/광고 연동

## 작업

- 랜딩 첫 화면에서 Deebi의 공동 개발 감성이 바로 보이는지 점검한다.
- dashboard의 내 방/함께하는 방 정보 위계를 정리한다.
- loading/empty/error/success 카피를 정리한다.
- 긴 login과 숫자 표시를 점검한다.

## 완료 조건

- [ ] 랜딩 핵심 문장이 잘 보인다.
- [ ] dashboard 정보 위계가 과밀하거나 비어 보이지 않는다.
- [ ] 주요 상태 카피가 있다.
- [ ] 모바일 360px에서 텍스트가 겹치지 않는다.

## 개발자 테스트

1. 모바일과 데스크톱에서 `/`를 확인한다.
2. dashboard empty/error/loading 상태를 확인한다.
