# P10-T01 — 디자인 토큰과 픽셀 UI 감사

## 목표

실제 기능 연결 후 UI가 여전히 Deebi의 픽셀 그래픽 감성을 유지하는지 감사한다.

## 변경 범위

- **해도 됨:** design token 정리, 색/타이포/간격 수정, 공통 컴포넌트 cleanup
- **하면 안 됨:** 신규 기능 추가, API 계약 변경

## 작업

- PF스타더스트 적용 범위를 확인한다.
- 픽셀 버튼/패널/배지/채팅 스타일이 일관된지 확인한다.
- 과한 SaaS 카드/gradient/glassmorphism 느낌을 줄인다.
- 색상 팔레트가 한 색조로만 읽히지 않게 점검한다.

## 완료 조건

- [ ] 주요 UI에 픽셀 폰트/토큰이 적용되어 있다.
- [ ] 버튼, 패널, 탭, 배지가 같은 디자인 언어를 쓴다.
- [ ] 과한 단색/gradient 테마가 아니다.
- [ ] style 변경이 layout shift를 만들지 않는다.

## 개발자 테스트

1. `/`, `/dashboard`, `/room/[code]`를 빠르게 훑어 디자인 톤을 확인한다.
2. 버튼/패널/채팅이 같은 제품처럼 보이는지 확인한다.
