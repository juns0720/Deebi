# P11-T03 — 핵심 도메인 로직 테스트 보강

## 목표

health, points, gacha, room 제한 같은 핵심 로직에 회귀 테스트를 보강한다.

## 변경 범위

- **해도 됨:** 순수 함수 테스트, API 단위 테스트, edge case 수정
- **하면 안 됨:** 새로운 보상/레벨 시스템 추가

## 작업

- health 0~100 범위 테스트를 확인한다.
- 신규 커밋만 포인트 지급되는지 테스트한다.
- gacha 포인트 부족/중복 환급 테스트를 확인한다.
- room 8명 상한과 message validation 테스트를 확인한다.

## 완료 조건

- [ ] health 범위 테스트가 있다.
- [ ] 중복 포인트 지급 방지 테스트가 있다.
- [ ] 포인트 부족 뽑기 테스트가 있다.
- [ ] 룸 메시지 입력 길이 테스트가 있다.
- [ ] 테스트 명령이 통과한다.

## 개발자 테스트

1. 프로젝트 테스트 명령을 실행한다.
2. 실패 케이스가 있다면 어떤 로직인지 확인한다.
