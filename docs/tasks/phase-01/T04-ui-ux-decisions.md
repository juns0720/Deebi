# P01-T04 — UI/UX 결정과 픽셀 감성 기준

## 목표

개발자와의 질의응답으로 확정된 Deebi의 UI/UX 방향을 문서화한다. 이 task가 끝나면 Phase 02에서 실제 화면을 만들 때 디자인 의사결정을 다시 묻지 않아도 된다.

## 변경 범위

- **해도 됨:** `docs/UI_UX.md`, `docs/UI_UX_QUESTIONS.md`, `docs/DECISIONS.md`, UI copy 문서
- **하면 안 됨:** 개발자가 직접 제작하기로 한 최종 캐릭터 디자인 임의 제작, 실제 광고 SDK 구현, 고급 채팅 기능 추가

## 작업

- 전체 UI를 픽셀 그래픽 감성으로 구현한다는 원칙을 기록한다.
- 기본 폰트를 PF스타더스트로 쓰는 결정을 기록한다.
- 개인 방, 공동 룸, 작업 테이블, 8명 상한, 방치 모드, 공동 룸 채팅 방향을 정리한다.
- 랜딩 핵심 문장 `각자 코딩해도, 같은 방에 있는 것처럼.`을 기록한다.
- 열린 UI/UX 질문과 확정된 답변을 분리한다.

## 완료 조건

- [ ] `docs/UI_UX.md`가 랜딩, 내 방, 함께하는 방, 공동 룸, 채팅, 방치 모드를 설명한다.
- [ ] `docs/UI_UX_QUESTIONS.md`가 확정/보류/추후 질문을 구분한다.
- [ ] `docs/DECISIONS.md`에 개발자 답변이 결정 로그로 누적되어 있다.
- [ ] PF스타더스트와 픽셀 UI 원칙이 Phase 02 구현 기준으로 명시되어 있다.

## 개발자 테스트

1. `docs/UI_UX.md`에서 Phase 02가 만들 화면을 확인한다.
2. `docs/DECISIONS.md`에 본인이 답한 UI/UX 결정이 빠지지 않았는지 확인한다.
3. 캐릭터 최종 디자인을 에이전트가 임의 제작하지 않도록 제한이 적혀 있는지 확인한다.
