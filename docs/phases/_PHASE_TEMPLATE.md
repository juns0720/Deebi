# Phase NN — 단계 이름

이 파일은 새 phase를 추가할 때 복사해서 사용하는 템플릿이다. 실제 진행 대상 phase에는 모든 항목을 구체적으로 채우고, 완료 조건은 사람이 검증 가능한 문장으로 작성한다.

## 목표

이 단계가 끝났을 때 사용자와 시스템 관점에서 무엇이 가능해지는지 한두 문장으로 쓴다.

## 변경 범위

- **해도 됨:** 이 단계에서 수정 가능한 파일, 모듈, 문서 범위를 적는다.
- **하면 안 됨:** 다른 task/phase 소관이거나 MVP 범위 밖인 작업을 적는다.

## 작업

- 구현해야 할 일을 항목별로 적는다.
- "무엇을"과 "왜"는 고정하고, "어떻게"는 에이전트가 코드베이스를 보고 결정할 여지를 남긴다.
- UI/UX 결정이 필요하면 `docs/UI_UX_QUESTIONS.md`를 참조하도록 적는다.

## 완료 조건

- [ ] 실행 또는 눈으로 검증 가능한 조건을 적는다.
- [ ] 자동 검증 명령이 있다면 포함한다.
- [ ] 문서 갱신 조건이 있다면 포함한다.

## 개발자 테스트

1. 개발자가 직접 따라 할 절차를 순서대로 적는다.
2. 기대 결과를 함께 적는다.
