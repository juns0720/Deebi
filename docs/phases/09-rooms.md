# Phase 09 — 룸

## 목표

룸 생성, 코드 참여, 멤버 캐릭터 그리드, 공동 룸 채팅, 룸 나가기를 구현한다. 이 단계가 끝나면 사용자는 룸 링크를 공유해 서로의 캐릭터 상태를 보고 같은 방 안에서 간단히 대화할 수 있다.

## 변경 범위

- **해도 됨:** room APIs, room message APIs, `/room/[code]`, room components, dashboard room 진입 UI, 공동 룸 텍스트 채팅 UI
- **하면 안 됨:** 친구 요청/수락, DM, 전역 채팅, 메시지 수정/삭제, 신고/차단, 멘션, 파일 첨부, 읽음 표시, 룸 삭제, 생성자 승인 시스템, 복잡한 권한 관리

## 작업

- room code 생성 함수를 구현한다.
- `POST /api/rooms`를 구현한다.
- `POST /api/rooms/join`을 구현한다.
- `GET /api/rooms/[code]`를 구현한다.
- `POST /api/rooms/[code]/leave`를 구현한다.
- `GET /api/rooms/[code]/messages`를 구현한다.
- `POST /api/rooms/[code]/messages`를 구현한다.
- dashboard의 `함께하는 방` 탭에 룸 생성/참여 진입 UI를 추가한다.
- dashboard의 `함께하는 방` 탭에 내가 참여한 룸 목록과 재입장 링크를 추가한다.
- `/room/[code]`에 멤버 캐릭터 그리드를 구현한다.
- 공동 룸은 큰 작업 테이블 중심의 공간으로 보이게 한다.
- 멤버 상태는 작은 라벨과 테이블 오브젝트/이펙트 조합으로 표현한다.
- 공동 룸 채팅 패널을 추가한다.
- 데스크톱에서 채팅은 룸 스테이지 오른쪽 320~360px 패널로 배치한다.
- 모바일에서 채팅은 하단 시트로 연다.
- 채팅은 최근 메시지 50개 조회와 1~300자 메시지 전송을 지원한다.
- 채팅 갱신은 MVP에서 짧은 폴링 또는 수동 갱신으로 충분하다.
- 공유 코드 복사 UI를 추가한다.
- 룸 광고 placeholder를 추가한다.

## 완료 조건

- [ ] 로그인 사용자가 룸을 만들 수 있다.
- [ ] 생성자가 자동으로 첫 멤버가 된다.
- [ ] code로 룸에 참여할 수 있다.
- [ ] 한 번 참여한 룸은 대시보드 `함께하는 방` 탭에서 다시 입장할 수 있다.
- [ ] 룸 멤버 수가 8명을 넘지 않는다.
- [ ] 8명 초과 참여 시 `ROOM_FULL` 오류를 반환한다.
- [ ] 이미 참여한 룸에 다시 참여해도 중복 row가 생기지 않는다.
- [ ] 룸 멤버가 health 순으로 정렬된다.
- [ ] 공동 룸 화면에 큰 작업 테이블 중심의 공간감이 있다.
- [ ] 각 멤버 카드에 login, health, today commit count, streak가 보인다.
- [ ] 멤버 활동 상태가 라벨과 작은 오브젝트/이펙트로 함께 표현된다.
- [ ] 룸 멤버는 공동 룸에서 텍스트 메시지를 보낼 수 있다.
- [ ] 룸 멤버는 최근 메시지를 볼 수 있다.
- [ ] 데스크톱에서 채팅 패널은 오른쪽에 있고 룸 스테이지와 캐릭터를 가리지 않는다.
- [ ] 모바일에서 채팅은 하단 시트로 열리고 닫을 수 있다.
- [ ] 룸 멤버가 아닌 사용자는 메시지를 조회하거나 작성할 수 없다.
- [ ] 빈 메시지와 300자 초과 메시지는 저장되지 않는다.
- [ ] 디비 말풍선은 공동 룸 채팅에 표시되지 않는다.
- [ ] 룸 나가기가 동작한다.
- [ ] 룸 나가기 후 내 룸 목록에서 제거된다.
- [ ] 룸 삭제/DM/친구 요청/친구 추가/신고/차단/파일 첨부/읽음 표시 기능은 없다.
- [ ] `npm run lint`, `npm run typecheck`, `npm run build`가 통과한다.

## 개발자 테스트

1. 사용자 A로 룸을 만든다.
2. room code 또는 `/room/[code]` URL을 확인한다.
3. 사용자 B로 로그인해 같은 룸에 참여한다.
4. 룸 페이지에서 두 사용자의 캐릭터가 보이는지 확인한다.
5. 사용자 A와 B가 같은 룸에서 메시지를 주고받을 수 있는지 확인한다.
6. 룸 멤버가 아닌 사용자로 메시지 API 접근이 거부되는지 확인한다.
7. 룸 나가기를 실행하고 멤버 목록에서 빠지는지 확인한다.
