## 단계 운영 원칙

- 한 번에 한 단계만 진행한다.
- Current Phase에 적힌 phase 파일 하나만 실행 대상으로 삼는다.
- phase 문서의 "하면 안 됨"을 절대 넘지 않는다.
- phase 완료 후 `docs/PROGRESS.md`를 `AWAITING REVIEW`로 바꾸고 멈춘다.
- 개발자가 테스트 통과를 알려주기 전 다음 단계로 넘어가지 않는다.

## 각 phase 문서의 공통 형식

모든 `NN-*.md`는 아래 섹션을 가진다.

1. **목표** — 이 단계가 끝나면 무엇이 되는가
2. **변경 범위** — 건드려도 되는 영역 / 건드리면 안 되는 영역
3. **작업** — 해야 할 일
4. **완료 조건** — 항목별 체크리스트
5. **개발자 테스트** — 사람이 직접 확인할 절차
6. **참조** — 관련 문서
