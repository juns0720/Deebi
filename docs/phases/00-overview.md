## 각 phase 문서의 공통 형식

모든 `NN-*.md`는 아래 섹션을 가진다. 에이전트는 이 형식을 신뢰한다.

1. **목표** — 이 단계가 끝나면 무엇이 되는가
2. **변경 범위** — 건드려도 되는 영역 / 건드리면 안 되는 영역
3. **작업** — 해야 할 일 (How는 에이전트 재량, What·Why는 고정)
4. **완료 조건** — 항목별 체크리스트. 전부 만족해야 단계 종료
5. **개발자 테스트** — 개발자가 직접 확인할 항목 (PROGRESS.md로 복사됨)
6. **참조** — 관련 SPEC/ARCHITECTURE 섹션
